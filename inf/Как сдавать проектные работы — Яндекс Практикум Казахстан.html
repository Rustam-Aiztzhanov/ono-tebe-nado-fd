Как подать проектную работу

Вы прочитали описание проекта. Перейдем к оставшимся шагам:
  1. Подключите свой аккаунт GitHub к Practice.
  2. Получите подготовку проекта и клонируйте ее репозиторий на свой компьютер.
  3. Выполнить проект на основе описания.
  4. Проверьте работу по контрольному списку.
  5. Отправьте код проекта на GitHub.
  6. Создать файл README.md
  7. Пройдите автотест и при необходимости внесите изменения в проект.
  8. Отправить проект на рассмотрение.
  9. Прочитайте отзывы рецензента.
  10. Если рецензент оставил комментарии, внесите изменения в проект и отправьте его на повторное рассмотрение.

Шаг 1. Подключите свой аккаунт GitHub к Praktikum

Чтобы отправить проектную работу на проверку, вам необходимо привязать свой аккаунт GitHub к платформе Yandex Praktikuma. Это нужно сделать только один раз:
  1. Нажмите на остальные уроки этой темы, чтобы перейти к уроку «Отправка проектной работы». Такая техническая особенность заключается в том, что аккаунт можно привязать только в интерфейсе отправки проекта. Но не забудьте вернуться к пропущенным урокам и изучить инструкцию. Напоминаем вам об этом.
  2. Нажмите на кнопку «Привязать учетную запись GitHub», вы будете перенаправлены в браузер, чтобы позволить нашей платформе собирать ваш код. Если вы не вошли в GitHub, вам будет предложено ввести имя пользователя и пароль. Посмотрите, как привязать учетную запись GitHub:

Шаг 2. Скачать подготовку проектной работы

  1. Если вы еще не установили Git на свой компьютер, сделайте это в разделе "Bash и Git. Основы". Также сгенерируйте SSH-ключ и добавьте его в настройки вашего аккаунта GitHub. Рекомендуем привыкнуть к использованию SSH, именно так работают с Git профессиональные разработчики.
  2. После того, как вы войдете в урок и отправите проектную работу, перейдите в свой аккаунт на GitHub. Вы должны увидеть, что репозиторий с первой проектной работой был скопирован в ваш аккаунт. Для проектной работы "Ono tebe nado" скопированный репозиторий будет иметь имя ono-tebe-nado-fd. Заготовки проектных работ следующих спринтов также будут скопированы вам в GitHub после перехода на урок и проекта, который будет отправлен.
  3. Важно:  проверьте, что вы клонируете работу в свой аккаунт. Ссылка должна выглядеть так:  git@github.com:ваш-аккаунт/ono-tebe-nado-fd.git. Вы не можете запустить изменения в чужом репозитории. Подождите, пока продукт появится в вашем репозитории после привязки к вашему аккаунту GitHub. Если прошло несколько часов, а заготовка так и не появилась, напишите ментору.
  4. Клонируем репозиторий на компьютер по SSH, выполнив команду в консоли  git clone, добавив к команде адрес репозитория. Адрес репозитория необходимо взять на GitHub:
Адрес репозитория на github для клонирования
После выполнения команды на вашем компьютере появится папка, в которой будет файл вашего репозитория.
Если на этом этапе у вас уже есть наработки — скопируйте их в папку клонированного проекта и сделайте коммит.
При этом обратите внимание, что вы не можете заменить папку .git, которая создается при клонировании репозитория, на другую или вручную вносить в нее какие-либо изменения. Папка .gitсодержит служебную информацию, необходимую для работы Git и привязки вашего локального репозитория к удаленному репозиторию на GitHub.

Шаг 3. Завершите задачу проекта и отправьте написанный код на GitHub

Пришло время работать с проектом:
  1. Внимательно изучите ТЗ на проектную работу. Продолжайте работу над проектом в локальном репозитории на вашем компьютере, выполнив задание. При работе над проектом обязательно сверяйтесь с контрольным списком, он поможет завершить работу с минимальными последующими правками.
  2. После выполнения проекта и проверки по контрольному списку сначала выполните команду, git add -Aчтобы git исправил все измененные файлы проекта.
  3. Затем зафиксируйте изменения с помощью команды git commit -m "текст", где краткое описание внесенных изменений вводится в виде текста.
  4. Теперь пришло время отправить код вашего проекта на GitHub. Для этого выполните команду в папке с репозиторием проекта git push.
  5. Убедитесь, что все изменения были отправлены: перейдите на GitHub, найдите репозиторий проекта и убедитесь, что содержимое файлов на вашем компьютере совпадает с содержимым файлов в репозитории проекта.
Если после выполнения каких-либо команд в консоли вы видите ошибку или информацию о том, что команда не может быть выполнена, и вы не можете решить проблему самостоятельно, обсудите эту ошибку с инструктором в чате группы.
